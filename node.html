# Nodejs安装

# nodejs下载

nodejs.org

下载选择扩展名为msi格式的，这是一个可执行文件，有向导安装方式。

# nodejs安装

  1. 下载完成后，双击安装包，开始安装，一直点next即可，安装路径默认在C:\Program Files下，也可以自定义修改。

  1. 自定义设置

Node.js runtime 表示运行环境 npm package manager 表示npm包管理器 online documentation shortcuts 在线文档快捷方式 Add to PATH 添加到环境变量

  1. 打开CMD窗口，执行命令node -v查看node版本

# npm包管理器

NPM是随同NodeJS一起安装的包管理工具，能解决NodeJS代码部署上的很多问题

# 查看版本

npm -v

# 常用命令

  1. 安装模块
npm install <Module Name>

例如

npm install express
  1. 全局安装模块
npm install <Module Name> -g

例如

npm install express -g
  1. 本地和全局安装区别
npm install express         # 本地安装
npm install express -g      # 全局安装
  1. 卸载模块
npm uninstall express
  1. 更新模块
npm update express

# package.json 包清单文件

  1. name - 包名。
  2. version - 包的版本号。
  3. description - 包的描述。
  4. homepage - 包的官网 url 。
  5. author - 包的作者姓名。
  6. contributors - 包的其他贡献者姓名。
  7. scripts - 运行脚本文件。
  8. dependencies - 依赖包列表。如果依赖包没有安装，npm 会自动将依赖包安装在 node_module 目录下。
  9. repository - 包代码存放的地方的类型，可以是 git 或 svn，git 可在 Github 上。
  10. main - main 字段指定了程序的主入口文件，require('moduleName') 就会加载这个文件。这个字段的默认值是模块根目录下面的 index.js。
  11. keywords - 关键字

可以通过npm init初始化包清单文件

# npm run

我们可以将常用的运行指令写在scripts下，给它取一个别名，这样可以很方便的运行它。

{
  "scripts": {
    "dev": "vuepress dev .",
    "build": "vuepress build ."
  }
}

运行的时候，就不需要写vuepress dev .，而只需要npm run dev

如果取名为start，可以使用npm start直接运行，省略run。

{
  "scripts": {
    "start": "vuepress dev ."
  }
}

# cnpm 淘宝镜像

淘宝 NPM 镜像是一个完整 npmjs.org 镜像，你可以用此代替官方版本(只读)，同步频率目前为 10分钟 一次以保证尽量与官方服务同步。

可以使用我们定制的 cnpm (gzip 压缩支持) 命令行工具代替默认的 npm

# 命令安装

npm install -g cnpm --registry=https://registry.npm.taobao.org

# 查看版本

cnpm -v
